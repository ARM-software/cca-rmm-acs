# Arm RHI ACS Testcase checklist
-----------------------------------------------------

This document presents the mapping of the rules in the RHI specification to the
test cases and the steps followed in the tests. This also provides the information
about the test coverage scenarios that are implemented in the current release of
ACS and the scenarios that are planned for the future releases.

| S.NO | TestName | Test Approach | Validated By ACS |
| --- | --- | --- | --- |
| 1 | rhi\_session\_version | 1. Realm invokes `RHI_HOST_CALL` with `IMM=0` and `GPRS[0]=RHI_SESSION_VERSION`.<br>2. Verify the host call completes successfully.<br>3. If `GPRS[0] == RHI_SMC_NOT_SUPPORTED`, treat the test as failed because this ABI is expected to be implemented.<br>4. Otherwise validate `GPRS[0] == RHI_SESSION_VER`. | Yes |
| 2 | rhi\_session\_features | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_SESSION_FEATURES`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_SESSION_SUPPORTED_MASK`.<br>4. Validate `GPRS[1] == RHI_CONN_MODE_SUPPORTED_MASK`. | Yes |
| 3 | rhi\_session\_open | 1. Invoke `RHI_SESSION_OPEN` with an invalid session ID → expect `RHI_SESS_INVALID_SESSION_ID`.<br><br>2. Invoke `RHI_SESSION_OPEN` with unsupported connection type `RHI_CONN_MODE_BLOCKING \| RHI_CONN_MODE_NON_BLOCKING` → expect `RHI_SESS_CONNECTION_TYPE_NOT_SUPPORTED`.<br><br>3. Invoke `RHI_SESSION_OPEN` with `RHI_CONN_MODE_BLOCKING` → expect `RHI_SESS_SUCCESS`, `GPRS[2] = RHI_HSS_CONNECTION_ESTABLISHED`, and a valid returned session ID.<br><br>4. Invoke `RHI_SESSION_OPEN` again with the current established session ID → expect `RHI_SESS_INVALID_STATE_FOR_OPERATION`.<br><br>5. Invoke `RHI_SESSION_CLOSE` after the blocking-open path → expect `RHI_SESS_SUCCESS`.<br><br>6. Invoke `RHI_SESSION_OPEN` with `RHI_CONN_MODE_NON_BLOCKING` → expect `RHI_SESS_SUCCESS` with `GPRS[2] = RHI_HSS_CONNECTION_IN_PROGRESS` and save the returned session ID.<br><br>7. Invoke `RHI_SESSION_OPEN` in non-blocking mode with an invalid session ID → expect `RHI_SESS_INVALID_SESSION_ID`.<br><br>8. Reinvoke `RHI_SESSION_OPEN` with the current session ID to poll completion → expect `RHI_SESS_SUCCESS` and `GPRS[2] = RHI_HSS_CONNECTION_ESTABLISHED`.<br><br>9. Invoke `RHI_SESSION_CLOSE` after the non-blocking-open path → expect `RHI_SESS_SUCCESS`.<br><br>10. Invoke `RHI_SESSION_CLOSE` again to verify repeated close handling. | Yes |
| 4 | rhi\_session\_close | 1. Open a session in `RHI_CONN_MODE_NON_BLOCKING` and capture the returned session ID.<br>2. Reinvoke `RHI_SESSION_OPEN` with that session ID to advance the connection to the established state.<br>3. Invoke `RHI_SESSION_CLOSE` with an invalid session ID; expect `RHI_SESS_INVALID_SESSION_ID`.<br>4. Invoke `RHI_SESSION_CLOSE` with the valid established session ID; expect `RHI_SESS_SUCCESS` and `GPRS[2]=RHI_HSS_CONNECTION_CLOSE_IN_PROGRESS`.<br>5. Reinvoke `RHI_SESSION_CLOSE` with the same session ID to poll completion; expect `RHI_SESS_SUCCESS` and `GPRS[2]=RHI_HSS_SESSION_UNCONNECTED`. | Yes |
| 5 | rhi\_session\_send | 1. Realm prepares a shared buffer and fills it with test data.<br>2. Open a session in `RHI_CONN_MODE_BLOCKING`; expect `RHI_HSS_CONNECTION_ESTABLISHED` and a valid session ID.<br>3. Invoke `RHI_SESSION_SEND` with valid session ID, aligned IPA buffer, `SEND_DATA_LENGTH`, and offset 0; expect `RHI_SESS_SUCCESS`, `GPRS[2]=RHI_HSS_IO_COMPLETE`, and `GPRS[3]=SEND_DATA_LENGTH`.<br>4. Close the blocking session and expect `RHI_HSS_SESSION_UNCONNECTED`.<br>5. Invoke `RHI_SESSION_SEND` with an invalid session ID; expect `RHI_SESS_INVALID_SESSION_ID`.<br>6. Open a session in `RHI_CONN_MODE_NON_BLOCKING`; expect `RHI_HSS_CONNECTION_IN_PROGRESS`.<br>7. Invoke `RHI_SESSION_SEND` while the connection is still in progress; expect `RHI_SESS_INVALID_STATE_FOR_OPERATION`.<br>8. Reinvoke `RHI_SESSION_OPEN` with the same session ID to poll until `RHI_HSS_CONNECTION_ESTABLISHED`.<br>9. Invoke `RHI_SESSION_SEND` with a misaligned IPA buffer; expect `RHI_SESS_ACCESS_FAILED`.<br>10. Invoke `RHI_SESSION_SEND` with aligned IPA in non-blocking mode; first expect `RHI_HSS_IO_IN_PROGRESS`, then reinvoke until `RHI_HSS_IO_COMPLETE` with `GPRS[3]=SEND_DATA_LENGTH`.<br>11. Invoke `RHI_SESSION_CLOSE` in non-blocking mode; first expect `RHI_HSS_CONNECTION_CLOSE_IN_PROGRESS`, then reinvoke until `RHI_HSS_SESSION_UNCONNECTED`. | Yes |
| 6 | rhi\_session\_receive | 1. Realm prepares a shared receive buffer and builds the expected byte pattern for data validation.<br>2. Open a session in `RHI_CONN_MODE_BLOCKING` and expect `RHI_HSS_CONNECTION_ESTABLISHED` with a valid session ID.<br>3. Invoke `RHI_SESSION_RECEIVE` with buffer IPA = 0, buffer size = 0, and offset = 0 to query the required size; expect `RHI_SESS_SUCCESS`, `GPRS[2]=RHI_HSS_BUFFER_SIZE_DETERMINED`, and returned length = `RECEIVE_DATA_LENGTH`.<br>4. Invoke `RHI_SESSION_RECEIVE` with aligned IPA and buffer size 0; expect `RHI_SESS_SUCCESS`, `GPRS[2]=RHI_HSS_IO_COMPLETE`, and bytes received = 0.<br>5. Invoke `RHI_SESSION_RECEIVE` with aligned IPA and `PAGE_SIZE`; expect `RHI_SESS_SUCCESS`, `GPRS[2]=RHI_HSS_IO_COMPLETE`, and bytes received = `RECEIVE_DATA_LENGTH`, then compare the received buffer against the expected pattern.<br>6. Close the blocking session and expect `RHI_HSS_SESSION_UNCONNECTED`.<br>7. Invoke `RHI_SESSION_RECEIVE` with an invalid session ID; expect `RHI_SESS_INVALID_SESSION_ID`.<br>8. Open a session in `RHI_CONN_MODE_NON_BLOCKING` and expect `RHI_HSS_CONNECTION_IN_PROGRESS`.<br>9. Invoke `RHI_SESSION_RECEIVE` while the connection is still in progress; expect `RHI_SESS_INVALID_STATE_FOR_OPERATION`.<br>10. Reinvoke `RHI_SESSION_OPEN` with the same session ID to poll until `RHI_HSS_CONNECTION_ESTABLISHED`.<br>11. Invoke `RHI_SESSION_RECEIVE` with a misaligned IPA buffer; expect `RHI_SESS_ACCESS_FAILED`.<br>12. Invoke `RHI_SESSION_RECEIVE` with aligned IPA in non-blocking mode; first expect `RHI_HSS_IO_IN_PROGRESS`, then reinvoke until `RHI_HSS_IO_COMPLETE` with bytes received = `RECEIVE_DATA_LENGTH`, and validate the received buffer contents.<br>13. Invoke `RHI_SESSION_CLOSE` in non-blocking mode; first expect `RHI_HSS_CONNECTION_CLOSE_IN_PROGRESS`, then reinvoke until `RHI_HSS_SESSION_UNCONNECTED`. | Yes |
| 7 | rhi\_fal\_version | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_FAL_VERSION`.<br>2. Verify the host call completes successfully.<br>3. If `GPRS[0] == RHI_SMC_NOT_SUPPORTED`, treat the test as failed because this ABI is expected to be implemented.<br>4. Otherwise validate `GPRS[0] == RHI_FAL_VER`. | Yes |
| 8 | rhi\_fal\_features | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_FAL_FEATURES`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_FAL_SUPPORTED_MASK`. | Yes |
| 9 | rhi\_fal\_get\_size | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_FAL_GET_SIZE`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_FAL_LOG_SIZE`. | Yes |
| 10 | rhi\_fal\_read | 1. Realm prepares an aligned shared buffer and constructs the expected FAL byte pattern.<br>2. Invoke `RHI_FAL_READ` with a misaligned IPA (`buffer_ipa + 1`); expect `RHI_FAL_ACCESS_FAILED`.<br>3. Invoke `RHI_FAL_READ` with aligned IPA and `PAGE_SIZE`; expect `RHI_FAL_SUCCESS`, `GPRS[1]=RHI_FAL_LOG_SIZE`, and `GPRS[2]=0`.<br>4. Compare the returned buffer contents against the expected FAL pattern.<br>5. Invoke `RHI_FAL_READ` again with the same aligned buffer; expect `RHI_FAL_SUCCESS`, `GPRS[1]=0`, and `GPRS[2]=0`, confirming the log is empty after the full read. | Yes |
| 11 | rhi\_da\_version | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_DA_VERSION`.<br>2. Verify the host call completes successfully.<br>3. If `GPRS[0] == RHI_SMC_NOT_SUPPORTED`, treat the test as failed because this ABI is expected to be implemented.<br>4. Otherwise validate `GPRS[0] == RHI_DA_VER`. | Yes |
| 12 | rhi\_da\_features | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_DA_FEATURES`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_DA_SUPPORTED_MASK`. | Yes |
| 13 | rhi\_da\_vdev\_continue | 1. Start a long-running DA operation by invoking `RHI_DA_VDEV_GET_INTERFACE_REPORT` on a valid `vdev_id`; expect `RHI_DA_ERROR_INCOMPLETE` and a non-zero continuation handle.<br>2. Invoke `RHI_DA_VDEV_CONTINUE` with an invalid `vdev_id`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>3. Invoke `RHI_DA_VDEV_CONTINUE` with the valid `vdev_id` but an invalid handle; expect `RHI_DA_ERROR_DEVICE`.<br>4. Invoke `RHI_DA_VDEV_CONTINUE` repeatedly with the valid `vdev_id` and saved handle; allow either `RHI_DA_ERROR_INCOMPLETE` or `RHI_DA_SUCCESS` on each iteration, and continue until final status `RHI_DA_SUCCESS`. | Yes |
| 14 | rhi\_da\_vdev\_get\_measurements | 1. Realm prepares a valid measurement-parameter structure in shared memory.<br>2. Invoke `RHI_DA_VDEV_GET_MEASUREMENTS` with an invalid `vdev_id`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>3. Invoke `RHI_DA_VDEV_GET_MEASUREMENTS` with an invalid params IPA of 0; expect `RHI_DA_ERROR_ACCESS_FAILED`.<br>4. Invoke `RHI_DA_VDEV_GET_MEASUREMENTS` with valid `vdev_id` and params IPA; allow `RHI_DA_ERROR_INCOMPLETE` or `RHI_DA_SUCCESS`, and if incomplete capture the non-zero continuation handle.<br>5. If the request is incomplete, invoke `RHI_DA_VDEV_CONTINUE` repeatedly with the saved handle until the final status becomes `RHI_DA_SUCCESS`. | Yes |
| 15 | rhi\_da\_vdev\_get\_interface\_report | 1. Invoke `RHI_DA_VDEV_GET_INTERFACE_REPORT` with an invalid `vdev_id`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>2. Invoke `RHI_DA_VDEV_GET_INTERFACE_REPORT` with a valid `vdev_id`; allow `RHI_DA_ERROR_INCOMPLETE` or `RHI_DA_SUCCESS`, and if incomplete capture the non-zero continuation handle.<br>3. If the request is incomplete, invoke `RHI_DA_VDEV_CONTINUE` repeatedly with the saved handle until the final status becomes `RHI_DA_SUCCESS`. | Yes |
| 16 | rhi\_da\_vdev\_set\_tdi\_state | 1. Invoke `RHI_DA_VDEV_SET_TDI_STATE` with an invalid `vdev_id` and `RHI_DA_TDI_CONFIG_LOCKED`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>2. Invoke `RHI_DA_VDEV_SET_TDI_STATE` with a valid `vdev_id` but an invalid TDI state value; expect `RHI_DA_ERROR_INVALID_OBJECT`.<br>3. Invoke `RHI_DA_VDEV_SET_TDI_STATE` with a valid `vdev_id` and `RHI_DA_TDI_CONFIG_LOCKED`.<br>4. Allow either `RHI_DA_ERROR_INCOMPLETE` or `RHI_DA_SUCCESS`.<br>5. If `RHI_DA_ERROR_INCOMPLETE` is returned, verify a non-zero continuation handle in `GPRS[1]`.<br>6. Invoke `RHI_DA_VDEV_CONTINUE` with the same `vdev_id` and returned handle.<br>7. Expect final status `RHI_DA_SUCCESS`. | Yes |
| 17 | rhi\_da\_vdev\_p2p\_unbind | 1. Invoke `RHI_DA_VDEV_P2P_UNBIND` with invalid source and destination `vdev_id` values; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>2. Invoke `RHI_DA_VDEV_P2P_UNBIND` with two valid peer VDEV IDs; expect `RHI_DA_ERROR_INCOMPLETE` and capture the continuation handle.<br>3. Invoke `RHI_DA_VDEV_CONTINUE` with the first VDEV ID and the saved handle; expect `RHI_DA_SUCCESS`.<br>4. Invoke `RHI_DA_VDEV_CONTINUE` with the second VDEV ID and the same handle; expect `RHI_DA_SUCCESS`. | No |
| 18 | rhi\_da\_vdev\_abort | 1. Invoke `RHI_DA_VDEV_ABORT` with an invalid `vdev_id`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>2. Invoke `RHI_DA_VDEV_ABORT` with a valid `vdev_id` provided by the host.<br>3. Expect `RHI_DA_SUCCESS`. | Yes |
| 19 | rhi\_da\_object\_size | 1. Invoke `RHI_DA_OBJECT_SIZE` with an invalid `vdev_id` and object type `RHI_DA_DEV_VCA`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>2. Invoke `RHI_DA_OBJECT_SIZE` with a valid `vdev_id` and an invalid object type value; expect `RHI_DA_ERROR_INVALID_OBJECT`.<br>3. Invoke `RHI_DA_OBJECT_SIZE` with a valid `vdev_id` and unsupported object type `RHI_DA_DEV_EXTENSION_EVIDENCE`; expect `RHI_DA_ERROR_OBJECT_UNSUPPORTED`.<br>4. Invoke `RHI_DA_OBJECT_SIZE` with a valid `vdev_id` and object type `RHI_DA_DEV_VCA`; expect `RHI_DA_SUCCESS` and a non-zero size in `GPRS[1]`. | Yes |
| 20 | rhi\_da\_object\_read | 1. Realm prepares an aligned shared buffer for object data.<br>2. Invoke `RHI_DA_OBJECT_READ` with an invalid `vdev_id`; expect `RHI_DA_ERROR_INVALID_VDEV_ID`.<br>3. Invoke `RHI_DA_OBJECT_READ` with a valid `vdev_id` but an invalid object type value; expect `RHI_DA_ERROR_INVALID_OBJECT`.<br>4. Invoke `RHI_DA_OBJECT_READ` with unsupported object type `RHI_DA_DEV_EXTENSION_EVIDENCE`; expect `RHI_DA_ERROR_OBJECT_UNSUPPORTED`.<br>5. Invoke `RHI_DA_OBJECT_READ` with invalid buffer IPA 0; expect `RHI_DA_ERROR_ACCESS_FAILED`.<br>6. Invoke `RHI_DA_OBJECT_READ` with a valid buffer IPA but invalid offset/length combination (`max_len=0x10`, `offset=0x20`); expect `RHI_DA_ERROR_INVALID_OFFSET`.<br>7. Invoke `RHI_DA_OBJECT_READ` with valid `vdev_id`, object type `RHI_DA_DEV_VCA`, aligned buffer IPA, `PAGE_SIZE`, and offset 0; expect `RHI_DA_SUCCESS` and a non-zero byte count in `GPRS[1]`. | Yes |
| 21 | rhi\_hostconf\_version | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_HOSTCONF_VERSION`.<br>2. Verify the host call completes successfully.<br>3. If `GPRS[0] == RHI_SMC_NOT_SUPPORTED`, treat the test as failed because this ABI is expected to be implemented.<br>4. Otherwise validate `GPRS[0] == RHI_HOSTCONF_VER`. | Yes |
| 22 | rhi\_hostconf\_features | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_HOSTCONF_FEATURES`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_HOSTCONF_SUPPORTED_MASK`. | Yes |
| 23 | rhi\_hostconf\_get\_ipa\_change\_alignment | 1. Realm invokes `RHI_HOST_CALL` with `GPRS[0]=RHI_HOSTCONF_GET_IPA_CHANGE_ALIGNMENT`.<br>2. Verify the host call completes successfully.<br>3. Validate `GPRS[0] == RHI_HOSTCONF_IPA_CHANGE_ALIGNMENT`. | Yes |
